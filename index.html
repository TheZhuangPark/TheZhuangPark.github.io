Nearest Neighbor Methods

Posted on September 14, 2018

##Nearest Neighbor Methods##

###political affiliation###

very easy to understand
approach 1:
first initialize all training examples (xi,yi)
Classify a new test point x(test) by finding x(test) closest neighbor xi, and classify x(test) with label yi.

comparing with eager algorithms, this is a lazy means learning that occur till test example is presented.

###distance###
there are several ways to measure the distance bwteen training point x-trn and test point x-tst.
1 Euclidean distance
2 Manhattan distance
3 Minkowski distance
4 Mahalanobis distance(important)
5 weighted distance
etc, we can produce Voronoi diagram to better discribes zone of control

tips: different measures of closeness produce different diagrams.

mahalanobis distance is a useful measure of closeness between two point with n dimensions.It should deserve more discussion.

###problem1:range###
definetely when we gather training data.we will see many features value with diferent range.
feature with lager range will become more important.(i don’t know why?)
But we can normalize feature value, making them share the same range.
here is the math.

###problem2:irrelevant inputs###
irrelevant inputs mean noise feature. noise feature add random perturbations to classifier.

three approaches can handle this issue.
1 feature weighting:weight the feature then use a weighted-distance as distance metric (don’t know how to weight the feature,how to calculate each weight)

2 metric learing: learn a metric(what metric should we learn?) use mahalanobis distance (should have more discussion)

3:smoothing:Find k nearest neighbors and have them vote for label, multiple neighbors can reduce the effects of noise.

compute distance between all trn-point and new tst-point take O(nd) time. With n data set and d dimension.

###k-Nearest Neighbor###
approach2:
1 initialize: store all trn examples (xi,yi)
2 find k trn-point xi such that xi are cloest to x-tst, and classify x-tst with majority label from neighbors set.

####problem a: selecting k####
when we got k=1 bias is low variance is high. when we got k=7 bias is high variance is low. there is a balance of selecting k.

k-nn can easily handle multi-class data
like
1 handwriting recognition
2 satellite scene image analysis
3 EKG pattern classification

####problem b: nearest neighbors are too far away####
we can expect some very far away neighbors have the same weight of vote as neighbors that are closer to test point.
we can set weight wi=1/D(Xi,X-tst)

####problem c: irrelevant feature####
we can weight each feature based on its’ ability to reduce classification error.
(what’s mutual information)
or Metric learing, use Mahalanobis
distance metric.

###Curse of Dimensionality###
just to remember as dimension increase.
1D we should go a distance of k/num
2D we should go a distance of sqrt(k/num)
3D …etc
So you konw its’ disability, weakpoint

###k-Dimensional trees###
this is a space-partitioning data structure for organizing points in a k-dimensional space.

non-leaf node: axis-aligned hyperplane that splits the space into two parts

leaf node: a k-dimensional point
The idea is that if new points to classify are evenly distributed
throughout the space, then the expected cost of classification is approximately (dlogn) operations, much smaller than (nd) operations.

###Summery###
advantages:
•Model is extremely simple and intuitive
• Very easy to implement, can be very fast in practice
• Flexible decision boundaries
• Variable-sized hypothesis space

disadvantages:
• Distance function must be carefully chosen
• Irrelevant features have a high impact and must be handled/removed
• Use feature weighting or metric learning
• Typically cannot handle high-dimensional spaces
• Memory and classification-time costs grow with dimensionality
• Use specialized data structures such as k-d trees to efficiently find
nearest neighbors
